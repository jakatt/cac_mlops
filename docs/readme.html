MLOps · Accidents routiers

Vue d'ensemble — CAC MLOps

Système MLOps de prédiction de gravité d'accidents routiers — pipeline automatisé de données à la production · versioning intégral · monitoring temps réel

🎯 Le projet en bref

Objectif : prédire si un accident de la route impliquera une victime grave (classification binaire grav ∈ {0, 1}) à partir de 27 features (conditions météo, type de voie, heure, caractéristiques de l'usager et du véhicule).

Source de données : données officielles ONISR (Observatoire National Interministériel de la Sécurité Routière) téléchargées depuis data.gouv.fr via API — 4 fichiers CSV par année, ~55 000 accidents/an. Périmètre actuel : 2021 → 2024.

Ce que démontre le projet :

  • Ingérer une nouvelle année de données de manière fiable et automatisée
  • Détecter automatiquement tout changement de format source (Pandera — 3 niveaux)
  • Versionner chaque évolution : données (DVC) · code (Git) · modèle (MLflow)
  • Benchmarker 3 algorithmes (RF / XGBoost / LightGBM) et sélectionner le champion
  • Déployer sans interruption de service (gate manuelle + rollback automatique)
  • Surveiller le comportement du modèle en production (drift Evidently + Grafana)
  • Sécuriser l'accès aux outils d'administration (Tailscale VPN mesh)
Modèle en production : lgbm_accidents@Production · cumul 2021+2022+2023 · 27 features  ·  Métriques : F1 = 0.664 · AUC = 0.839 · Recall = 0.631 · Acc = 0.783

⚡ Les 3 triggers de mise en production
TriggerDéclencheurAction
T1 — Nouvelle dataPrefect cron lundi 8h · détection ONISR automatiqueETL → validation → benchmark 3 algos → gate manuelle → promote → test-api → Kapsule
T2 — Nouveau codePush vers main (hors blueprint)CI → build images → VPS prépare → gate manuelle → compose up → test-api → Kapsule
T3 — Nouveau modèle DSPR avec config/model_params.yml uniquementBenchmark avec nouveaux hyperparamètres → si champion : gate manuelle → promote → test-api → Kapsule

Gate manuelle (tous triggers) : pause_flow_run() Prefect natif — validée via l'onglet Cockpit du cockpit Gradio (GO/STOP) ou Prefect UI. Intervient avant toute interruption VPS.
