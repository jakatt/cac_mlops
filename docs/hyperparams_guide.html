Data Science — Blueprint

Guide des hyperparamètres

Référence complète des hyperparamètres configurables pour chacun des trois algorithmes. Ces valeurs sont consignées dans config/model_params.yml après expérimentation locale, puis transmises au réentraînement de production via le Trigger 3.

Workflow DS (Trigger 3)
  1. Expérimenter localement : GridSearch / RandomizedSearch / Optuna — MLflow tracking
  2. Consigner les meilleurs params dans config/model_params.yml
  3. git commit + PR → main → déclenche automatiquement update-model-flow
  4. Benchmark RF / XGBoost / LightGBM → gate KPI → gate manuelle → promote @Production
RF
Random Forest — sklearn.ensemble.RandomForestClassifier
Ensemble de CART, bagging + feature sampling. Robuste, peu sensible aux hyperparamètres, bon point de départ.
Complexité des arbres
ParamètreTypeValeur actuelleEffet & conseils
n_estimators int 200 Nombre d'arbres. ↑ = meilleure perf + variance réduite, ↑ temps linéaire. Diminuer si durée trop longue (≥100 en prod).
max_depth int | null null null = illimité (conseillé pour RF — la profondeur est naturellement contrainte par min_samples_leaf). Fixer une valeur (ex: 15–30) uniquement si mémoire critique.
min_samples_split int 2 Nœud splitté uniquement si ≥ N échantillons. ↑ = moins de splits, arbres plus simples. Tester 5–20 si surapprentissage.
min_samples_leaf int 1 Min échantillons dans une feuille. ↑ = arbres plus lisses, moins de surfit. Tester 2–10 sur ce dataset.
max_features str | float "sqrt" Features évaluées à chaque split. "sqrt" = √(nb_features) (défaut sklearn, 27 features → 5/6). "log2" = plus rapide mais moins expressif. Float 0–1 pour contrôle fin (ex: 0.5 = 50%).
Échantillonnage
ParamètreTypeValeur actuelleEffet & conseils
bootstrap bool true Tirage avec remplacement (bagging). true = RF classique. false = pasting (chaque arbre sur tout le dataset, moins de diversité).
max_samples float | null null Fraction du dataset par arbre (nécessite bootstrap=true). null = 100%. Ex: 0.8 → 80% des lignes par arbre, accélère et diversifie.
Critère & classes déséquilibrées
ParamètreTypeValeur actuelleEffet & conseils
criterion str "gini" Critère de qualité des splits. "gini" (défaut, rapide) ou "entropy" (information gain, légèrement plus lent). En pratique très peu de différence sur de grands datasets.
class_weight str | null null Pondération des classes. "balanced" = poids inversement proportionnel à la fréquence → utile si le ratio blessés graves / indemnes est déséquilibré. "balanced_subsample" recalcule à chaque arbre.
Params infra (non modifiables via blueprint) : n_jobs=-1 (parallélisme total) · random_state=42 (reproductibilité)
XGB
XGBoost — xgboost.XGBClassifier
Gradient boosting séquentiel avec régularisation L1/L2 intégrée. Très performant, nombreux leviers de contrôle du surapprentissage.
Complexité & apprentissage
ParamètreTypeValeur actuelleEffet & conseils
n_estimators int 300 Nombre de boosting rounds. ↑ avec ↓ learning_rate pour meilleure perf. Surveiller overfitting via early stopping en local.
max_depth int 5 Profondeur max par arbre. Plage typique : 3–8. ↑ = plus expressif, risque surfit. Pour XGBoost contrairement à RF, il faut fixer une valeur (pas null).
learning_rate float 0.05 Taux de shrinkage (eta). ↓ = plus robuste, nécessite ↑ n_estimators. Typiquement 0.01–0.3. Couple {0.05, 300} ou {0.01, 1000} selon contrainte temps.
Sous-échantillonnage (anti-surapprentissage)
ParamètreTypeValeur actuelleEffet & conseils
subsample float 1.0 Fraction des lignes par arbre. Ex: 0.8 = 80% des données → réduit le surfit et accélère. Plage recommandée : 0.6–1.0.
colsample_bytree float 1.0 Fraction des features par arbre. Ex: 0.8 → diversifie les arbres, réduit corrélation. Très efficace sur nos 27 features. Plage : 0.5–1.0.
Régularisation
ParamètreTypeValeur actuelleEffet & conseils
min_child_weight int 1 Poids minimum (somme des hessiens) dans une feuille. ↑ = feuilles moins spécifiques, freine les splits rares. Tester 3–10 si surfit.
gamma float 0 Gain minimum pour accepter un split. 0 = aucune contrainte. ↑ = arbres plus conservateurs (moins de splits). Tester 0.1–1.0.
reg_alpha float 0 Régularisation L1 (lasso) sur les poids des feuilles. ↑ = sparsité, poids proches de 0. Utile si peu de features vraiment informatives.
reg_lambda float 1 Régularisation L2 (ridge) sur les poids des feuilles. ↑ = poids plus petits et lisses. Défaut = 1 (actif). Tester 0.5–5.
Déséquilibre de classes
ParamètreTypeValeur actuelleEffet & conseils
scale_pos_weight float 1 Ratio classe négative / positive. Si n_neg/n_pos ≈ 3 → mettre 3. Améliore recall sur la classe minoritaire (blessures graves). Vérifier via y_train.value_counts().
Params infra (non modifiables via blueprint) : n_jobs=-1 · random_state=42 · verbosity=0 · eval_metric="logloss"
LGBM
LightGBM — lightgbm.LGBMClassifier
Gradient boosting leaf-wise (vs. level-wise XGBoost). Plus rapide, très efficace sur les grands datasets. Champion actuel @Production.
Complexité & apprentissage
ParamètreTypeValeur actuelleEffet & conseils
n_estimators int 300 Nombre de boosting rounds. Coupler avec learning_rate : ↓ lr → ↑ n_estimators.
num_leaves int 63 Paramètre de complexité principal de LightGBM (leaf-wise). Max feuilles par arbre. Valeur théorique : 2^max_depth. ↑ = plus expressif, risque surfit. Typiquement 31–255. Toujours garder num_leaves < 2^max_depth si max_depth fixé.
max_depth int -1 -1 = illimité (contrôlé par num_leaves). Fixer uniquement pour contraindre durement la profondeur (ex: 10–15).
learning_rate float 0.05 Taux de shrinkage. ↓ = plus robuste, ↑ n_estimators. Même logique que XGBoost.
Sous-échantillonnage
ParamètreTypeValeur actuelleEffet & conseils
subsample float 1.0 Fraction des lignes par arbre. Nécessite subsample_freq > 0 pour être actif. Ex: 0.8 + subsample_freq=5.
subsample_freq int 0 Fréquence du bagging. 0 = désactivé. Ex: 5 → bagging tous les 5 arbres. Mettre > 0 pour activer subsample.
colsample_bytree float 1.0 Fraction des features par arbre. Actif sans condition. Ex: 0.8. Alias LightGBM : feature_fraction.
Régularisation
ParamètreTypeValeur actuelleEffet & conseils
min_child_samples int 20 Min données dans une feuille. Contrôle majeur du surfit pour LightGBM. ↑ = feuilles moins spécifiques. Tester 10–100 selon taille du dataset.
reg_alpha float 0.0 Régularisation L1 (lasso). ↑ = sparsité. Tester 0–1.
reg_lambda float 0.0 Régularisation L2 (ridge). ↑ = poids plus petits. Tester 0–10.
Classes déséquilibrées
ParamètreTypeValeur actuelleEffet & conseils
class_weight str | null null Pondération des classes. "balanced" = poids inversement proportionnel à la fréquence. Améliore recall sur la classe minoritaire. Peut être combiné avec subsample.
Params infra (non modifiables via blueprint) : n_jobs=-1 · random_state=42 · verbose=-1
