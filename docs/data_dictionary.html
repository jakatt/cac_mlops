⚙️ Notes de preprocessing importantes
Logique de fusion multi-usagers

Un accident peut impliquer plusieurs usagers. Pour obtenir une ligne par accident, on retient la victime présentant la gravité maximale (grav le plus élevé avant recodage). Les comptages nb_victim et nb_vehicules sont calculés avant ce dédoublonnage.

Valeurs manquantes — stratégie d'imputation

Colonnes obligatoires (toute ligne avec NaN supprimée) : catv, vma, secu1, obsm, atm

Colonnes imputées par mode sur X_train (appliqué à X_test avec les mêmes valeurs) : surf, circ, col, motor

Convention −1 et 0 : dans les fichiers ONISR, −1 = « non renseigné » et 0 peut signifier « sans objet » ou « non renseigné » selon la colonne. Ces deux valeurs sont converties en NaN avant imputation.
