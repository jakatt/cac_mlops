MLOps · Accidents routiers

Résumé exécutif

Synthèse complète du projet — données ONISR · modèle de classification binaire · infrastructure MLOps opérationnelle en production (2026-07)

📋 En deux phrases

Système MLOps complet de prédiction de gravité d'accidents routiers, entraîné sur les données officielles ONISR (data.gouv.fr) — années 2021 → 2024 disponibles. Le pipeline s'exécute de bout en bout chaque année lors de la publication de nouvelles données — avec versioning intégral (DVC · Git · MLflow) — et utilise la dernière année disponible comme source de données pour le monitoring drift Evidently (simulation production via scripts/simulate_production.py).

Année drift auto-détectée : quand les données 2025 seront disponibles (publiées ~juin 2027), elles deviendront automatiquement l'année drift et 2024 rejoindra le train set.

📊 Slide 1 — Flow annuel de mise à jour des données

Déclenché chaque année en juin, à la publication des données ONISR de l'année N-1.

Étape 1 — Validation du schéma (Pandera)
NiveauContrôleSi KO
Niveau 1Format fichier (encodage, séparateur, présence, non vide)❌ CRITICAL → STOP + alerte équipe · modèle année N-1 reste en production
Niveau 2Colonnes requises présentes, types corrects❌ CRITICAL → STOP + alerte équipe
Niveau 3Distributions dans les plages historiques⚠️ WARNING → log + pipeline continue
Étape 2 — Preprocessing (make_dataset.py)

Fusion des 4 tables → ~55 000 lignes × 27 features

Feature engineering, nettoyage, split temporel (N-1 ans train / dernière année test) — ratio ~55k lignes/an stable.

Étape 3 — Versioning DVC

dvc add → dvc push → Scaleway Object Storage s3://cac-mlops-data/dvc

Tag Git : data-v{N} — reproductibilité totale des données à chaque cycle.

Étape 4 — Entraînement (train_model.py)

Benchmark RF / XGBoost / LightGBM sur cumul 2021 → année N

Tracking : paramètres, métriques, modèle → MLflow run #{N}

Sélection du champion : meilleur F1 + gate KPI absolue + delta vs @Production

Étape 6 — Déploiement

Gate manuelle dans l'onglet Cockpit du cockpit Gradio (ou Prefect UI) → GO/STOP

Après GO : promotion @Production MLflow + restart API → test-api (6 tests) → rolling update Kapsule (si actif)

Interruption VPS : ~30-90 s (pendant docker compose up -d) — jamais avant la gate manuelle.

Kapsule K8s : zéro interruption (maxUnavailable:0).
