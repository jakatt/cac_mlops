4 gates — 1 rollback — 1 interruption

  • Gate 1 — Vérification source data.gouv : URL inaccessible ou fichier absent → flow Prefect stoppé + alerte
  • Gate 2 — Validation schéma Pandera CRITICAL : format, colonnes, volumétrie hors norme → ETL stoppé + alerte email SMTP
  • Gate 3 — KPI gate failed : aucun champion parmi RF/XGB/LGBM + alerte Grafana, @Production inchangé
  • Gate 4 — Smoke test timeout (deploy-vps-flow) : flow Prefect en échec + alerte Prefect
  • Rollback — alias MLflow @Production repointe vers la version précédente (Prefect UI ou CLI)
  • Interruption — API redémarre après promotion du nouveau modèle ≈30 s

3 gates — 1 rollback — 1 interruption

  • Gate 1 — CI lint/pytest fail : merge bloqué par branch protection + notification GitHub Actions
  • Gate 2 — Trivy CVE CRITICAL : job deploy annulé avant tout contact VPS + annotation erreur
  • Gate 3 — Smoke test timeout (120 s) : deploy bloqué + rollback images Docker déclenché
  • Rollback — smoke test fail + nouvelles images buildées : tags :rollback restaurés automatiquement via docker compose up
  • Interruption — redémarrages ciblés des conteneurs modifiés (nginx ≈1 s, gradio ≈5 s, api ≈30 s si image reconstruite)

4 gates — 1 rollback — 1 interruption

  • Gate 1 — CI lint/pytest fail : identique T2
  • Gate 2 — Trivy CVE CRITICAL : identique T2
  • Gate 3 — KPI gate failed (train_flow) : aucun champion, @Production inchangé + alerte Grafana
  • Gate 4 — Smoke test timeout : identique T2
  • Rollback — smoke test fail + new images → :rollback restaurées (images Docker)
  • Interruption — API redémarre pour charger le nouveau modèle depuis MLflow Registry ≈30 s
Alerte = email SMTP · notification GitHub Actions · alerte Grafana selon le contexte Rollback T1 : alias MLflow Registry · Rollback T2/T3 : images Docker
