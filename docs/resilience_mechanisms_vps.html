4 gates auto — 1 gate manuelle — 1 rollback — 1 interruption

  • Gate 1 Vérification source data.gouv : URL inaccessible ou fichier absent → flow Prefect stoppé + alerte
  • Gate 2 Pandera CRITICAL : format, colonnes, volumétrie hors norme → ETL stoppé + alerte email SMTP
  • Gate 3 KPI gate failed : aucun champion RF/XGB/LGBM → @Production inchangé + alerte Grafana
  • Smoke test pré-gate Santé du VPS actuel vérifiée avant ouverture de la gate (90 s sur /health) : si fail → flow stoppé immédiatement, état déjà dégradé, rien de déployé.
  • Gate manuelle Flow deploy-vps-flow mis en pause (pause_flow_run) : métriques du champion (f1/recall/auc/accuracy) affichées dans l'onglet Cockpit ou Prefect UI. Rien n'est encore appliqué en prod à ce stade. GO → promotion. STOP → run annulé, @Production inchangé, aucun impact.
  • Gate 4 Après GO : promotion @Production + redémarrage API (≈30 s) puis test-api (6 tâches) : si fail → rollback alias MLflow @Production + restart API + alerte Prefect

4 gates auto — 1 gate manuelle — 1 rollback — 1 interruption

  • Gate 1 CI lint/pytest fail → merge bloqué par branch protection + notification GitHub Actions
  • Gate 2 Trivy CVE CRITICAL → deploy annulé avant tout contact VPS + annotation erreur
  • Smoke test pré-gate Santé du VPS actuel vérifiée avant ouverture de la gate (90 s sur /health) : si fail → flow stoppé immédiatement, images déjà pull-ées mais aucun conteneur touché.
  • Gate manuelle Flow deploy-vps-flow mis en pause : SHA, image à builder, services à redémarrer affichés dans l'onglet Cockpit ou Prefect UI. Images déjà pull-ées mais pas encore appliquées. GO → docker compose up -d. STOP → run annulé, aucun conteneur touché.
  • Gate 3 Après GO : smoke test (90 s) une fois docker compose up -d --remove-orphans appliqué (redémarrage ciblé des services impactés) : si fail → rollback Docker :rollback + alerte Prefect
  • Gate 4 test-api (6 tâches) après le smoke test : si fail → rollback Docker :rollback, Kapsule non déclenché + alerte Prefect

4 gates auto — 1 gate manuelle — 1 rollback — 1 interruption

  • Gate 1 CI lint/pytest fail : identique T2
  • Gate 2 Trivy CVE CRITICAL : identique T2
  • Gate 3 KPI gate failed (train_flow) : aucun champion → @Production inchangé + alerte Grafana
  • Smoke test pré-gate Santé du VPS actuel vérifiée avant ouverture de la gate (90 s sur /health) : si fail → flow stoppé immédiatement, rien de déployé.
  • Gate manuelle Flow deploy-vps-flow mis en pause : métriques du nouveau blueprint + SHA/services à redémarrer affichés dans l'onglet Cockpit ou Prefect UI. Rien n'est encore appliqué en prod. GO → promotion + (si code impacté) docker compose up -d. STOP → run annulé, aucun impact.
  • Gate 4 Après GO : promotion @Production + redémarrage API (≈30 s), puis si code impacté docker compose up -d + smoke test (90 s), puis test-api (6 tâches) : si fail → rollback alias MLflow @Production ET/OU rollback Docker :rollback + alerte Prefect
